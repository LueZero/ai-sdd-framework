# 工作流：新功能開發

從零開始開發一個全新功能的完整流程。

## 適用情境

- 全新的功能需求
- 重大功能擴展
- 跨多個模組的功能開發

## 完整流程

```
Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6 → Phase 7
初始化     規格化     設計      實作      測試      審查      迭代
```

## 步驟詳解

### Step 1: 確認專案已初始化

如果是首次使用 SDD 流程，先執行 `/01-init-project` 建立專案上下文。

確認 `docs/sdd/project-context.md` 存在。

### Step 2: 撰寫功能規格

```
指令：/02-write-spec
Agent：@sdd-pm
輸入：需求描述
產出：docs/sdd/specs/SPEC-{編號}-{功能名}.md
```

**品質關卡**：規格書中的每個驗收條件都是可測試的？範圍界定清晰？

### Step 3: 系統設計

```
指令：/03-design-system
Agent：@sdd-architect
輸入：SPEC-{編號}
產出：docs/sdd/designs/DESIGN-{編號}-{功能名}.md
      docs/sdd/decisions/ADR-{編號}-{決策主題}.md（如有重大決策）
```

**品質關卡**：設計是否涵蓋規格的所有需求？元件職責清晰？

### Step 4: 實作

```
指令：/04-implement
Agent：@sdd-developer
輸入：SPEC-{編號} + DESIGN-{編號}
產出：原始碼
```

**品質關卡**：實作是否嚴格遵循規格？沒有加入規格外的功能？

### Step 5: 撰寫測試

```
指令：/05-write-tests
Agent：@sdd-developer
輸入：SPEC-{編號}
產出：測試程式碼
      docs/sdd/test-plans/TEST-{編號}-{功能名}.md
```

**品質關卡**：每個驗收條件都有對應測試？測試全部通過？

### Step 6: 程式碼審查

```
指令：/06-review
Agent：@sdd-reviewer
輸入：SPEC-{編號} + 實作程式碼 + 測試程式碼
產出：審查報告
```

**品質關卡**：沒有嚴重問題？規格符合度 100%？

### Step 7: 迭代修正

```
指令：/07-iterate
輸入：審查報告
產出：修正後的程式碼 + 更新的文件
```

**品質關卡**：所有審查問題已解決？測試仍全部通過？

## 完成條件

- [ ] 規格書狀態為「已完成」
- [ ] 所有驗收條件有對應的通過測試
- [ ] 審查報告判定為「通過」
- [ ] 相關文件已更新
- [ ] 程式碼已準備好合併/部署

## 預估時間分配

| 階段 | 佔比 |
|------|------|
| 規格 + 設計 | 30-40% |
| 實作 | 30-40% |
| 測試 + 審查 + 迭代 | 20-30% |
