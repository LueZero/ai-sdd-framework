# 工作流：規格變更管理（Change Request）

處理開發過程中發現需要修改規格的情況。

## 適用情境

- 實作過程中發現規格有遺漏或錯誤
- 設計過程中發現技術限制導致規格不可行
- 業務需求在開發中途改變
- 審查後發現規格根本問題
- 利害關係人要求調整功能範圍

## 核心原則

> **規格變更永遠先於程式碼變更。**
> 不存在「先改程式碼、之後再補規格」。

## 變更類型

| 類型 | 說明 | 範例 |
|------|------|------|
| **澄清（Clarification）** | 規格含糊、需要補充說明 | 驗收條件不夠具體 |
| **修正（Correction）** | 規格有錯誤 | 資料欄位型別定義有誤 |
| **擴展（Extension）** | 增加原規格未涵蓋的需求 | 新增一個驗收條件 |
| **縮減（Reduction）** | 移除或簡化部分需求 | 因技術限制調整功能範圍 |
| **重定向（Redirect）** | 業務方向改變，大幅調整 | 整個功能重新設計 |

## 步驟詳解

### Step 1: 識別變更需求

當你在任何階段發現需要修改規格時，**停止目前的實作**，先評估：

```
評估問題：
- 這個變更的範圍是什麼？
- 哪些已完成的工作受到影響？
- 影響層面：規格 / 設計 / 程式碼 / 測試
```

### Step 2: 撰寫變更請求

```
指令：/08-change-request
Agent：@sdd-pm（規格類變更）或 @sdd-architect（設計類變更）
產出：docs/sdd/changes/CR-{編號}-{描述}.md
```

使用 `sdd/templates/change-request.tmpl.md` 範本。

### Step 3: 影響評估

評估變更對已完成工作的影響：

```
影響地圖：
├── 規格書（docs/sdd/specs/SPEC-XXX.md）
│   └── 哪些驗收條件需要修改/新增/刪除？
├── 設計文件（docs/sdd/designs/DESIGN-XXX.md）
│   └── 哪些設計決策受到影響？
├── 程式碼
│   └── 哪些已實作的部分需要修改？
├── 測試
│   └── 哪些測試需要更新？
└── ADR
    └── 是否需要新增決策記錄？
```

### Step 4: 更新文件（先於程式碼）

按照以下順序更新文件：

1. **更新規格書**：修改驗收條件、資料契約、非功能需求
2. **更新設計文件**：如果設計受到影響
3. **新增 ADR**：如果有新的重大決策
4. **更新測試計劃**：反映新的驗收條件

### Step 5: 更新程式碼

文件更新完成後，才修改程式碼：

```
指令：/04-implement（重新引用更新後的規格）
Agent：@sdd-developer
```

### Step 6: 更新測試

```
指令：/05-write-tests（重新引用更新後的規格）
Agent：@sdd-developer
```

### Step 7: 重新審查

```
指令：/06-review（引用 CR-{編號}）
Agent：@sdd-reviewer
```

審查重點：
- 變更是否完整實施？
- 是否有遺漏更新的相關部分？
- 新的驗收條件都有測試覆蓋？

## 決策矩陣

根據變更的範圍，決定需要哪些步驟：

| 影響範圍 | 更新規格 | 更新設計 | 修改程式碼 | 更新測試 | 重新審查 |
|---------|---------|---------|----------|---------|---------|
| 只是文字澄清 | ✅ | — | — | — | — |
| 驗收條件調整 | ✅ | — | — | ✅ | ✅ |
| 資料結構變更 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 介面設計變更 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 架構重大調整 | ✅ | ✅ + ADR | ✅ | ✅ | ✅ |
| 功能範圍大幅調整 | 重新 Phase 2 | 重新 Phase 3 | 重新 Phase 4 | 重新 Phase 5 | ✅ |

## 版本控制建議

- 規格書修改後更新「最後更新」日期和「版本」
- 在規格書底部加入「變更歷史」章節
- 每個 CR 對應一個 Git commit，commit message 引用 `CR-{編號}`

## 完成條件

- [ ] 變更請求文件已建立
- [ ] 規格書已更新並標記版本
- [ ] 相關設計文件已更新
- [ ] 程式碼已按新規格修改
- [ ] 測試已更新且全部通過
- [ ] 重新審查完成
