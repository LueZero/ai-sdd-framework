# Phase 7: 迭代精煉

根據審查報告的回饋，進行修正和優化。

## 前置條件
- 審查報告已產出
- 知道需要修正的項目清單

## 步驟

### 1. 分類問題

將審查報告中的問題分類並排序：

```
優先順序：
1. 🔴 嚴重問題（安全漏洞、功能缺陷）
2. 🟡 警告問題（程式碼品質、效能）
3. 🟢 建議事項（可讀性、最佳實踐）
```

### 2. 逐項修正

針對每個問題：
1. 確認問題的根本原因
2. 規劃修正方案
3. 實施修正
4. 驗證修正是否解決問題
5. 確認修正沒有引入新問題

### 3. 更新測試

- 針對修正的問題補充測試
- 確認所有既有測試仍然通過
- 如有新的邊界情境，追加測試案例

### 4. 更新文件

如果修正涉及：
- 規格變更 → 更新 `docs/sdd/specs/` 中的規格書
- 設計變更 → 更新 `docs/sdd/designs/` 中的設計文件
- 新的技術決策 → 新增 ADR 至 `docs/sdd/decisions/`

### 5. 最終驗證

```
完成檢查清單：
- [ ] 所有 🔴 嚴重問題已修復
- [ ] 所有 🟡 警告問題已處理或記錄為 known issue
- [ ] 所有測試通過
- [ ] 文件已更新
- [ ] 規格中的所有驗收條件都有對應的通過測試
- [ ] 沒有引入新的安全問題
```
